## 5. Problems Found

### 🔴 P1 — Coordinator metrics: `active_sessions` label wrong

**File:** `src/coordinator/router.py:80`
```python
active_sessions.labels(node=decision.node_name).set(session_table.active_count)
```
This sets the **total** session count (`len(all_sessions)`) on the specific node label. Both RTX and P100 labels show the same value (e.g. 16). It should track per-node counts.

**Impact:** Grafana active_sessions panel is misleading — shows same number for both nodes.

### 🔴 P1 — P100 agent HTTP debug endpoint crashed

The P100 agent's HTTP server (`/debug`, `/metrics`) stopped responding mid-run. Connection was reset by peer. The RPC health check continued working (coordinator used TCP health, not HTTP). Fixed by restarting the container.

**Root cause:** Unknown — possibly an unhandled exception in `StartDebugEndpointAsync`. The `app.RunAsync()` runs as a fire-and-forget task with no error handling.

**Impact:** Metrics and debug endpoints unavailable for P100 agent. Prometheus scrape failed.

### 🟡 P2 — RTX slots stuck `isProcessing: true`

Both llama-server slots showed `isProcessing: true` even when idle (`n_past: 0`, no active task). This happened after a large prefill completed but the slot was never marked idle.

**Impact:** Inflates `slots_total - slots_idle` → makes RTX look fully loaded when it's not → new sessions go to P100 sooner than expected.

### 🟡 P2 — Session table unbounded (no eviction)

No session eviction runs. After the earlier e2e test run, 78 stale sessions remained in the coordinator's `session_table`, many hours old. The `session_idle_timeout_s = 3600` config exists but no eviction loop consumes it.

**Code gap:** `session_table.py` has `evict_stale()` method but it's never called.

**Impact:** Memory leak on coordinator. Session table grows forever.

### 🟡 P2 — `n_past` never updated

All sessions show `n_past: 0`. The coordinator never reads slot state back after a completion. The `session_table.update_n_past()` is never called from the router.

**Impact:** The `n_past_guard_triggered` logic on line 100-112 is useless — it resets `n_past` from 0 to 0 forever. Cannot track KV cache progress per session.

### 🟡 P3 — Agent gauges `slots_idle` / `llama_healthy` defined but not set

`AgentMetrics.cs` defines: `Gauge SlotsIdle`, `Gauge LlamaHealthy`. These are set in the `/debug` HTTP endpoint handler but NOT in the regular RPC health check path (`HandleNodeHealthAsync`). So the gauges are always 0/unset unless someone hits `/debug`.

**Impact:** Prometheus scrapes `/metrics` and gets HELP lines but no data for these metrics.

### 🟡 P3 — Coordinator agent/store histograms all zero

The coordinator exports `hydra_agent_save_duration_seconds`, `hydra_agent_restore_duration_seconds`, `hydra_store_ops_total`, `hydra_store_bytes_transferred_total` — but all counters and buckets are 0. The coordinator never increments these metrics through the RPC layer.

`hydra_migrations_total` is 0. No session has ever been migrated between GPUs in production. The `/sessions/{id}/migrate` endpoint exists but has never been called outside tests.
